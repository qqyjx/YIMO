## 功能演示 7 步（16 分钟）

### STEP 1 首页 · 2 分钟

**点**：浏览器打开 `http://localhost:5000/`

**说**：
> "系统首页，三个核心指标：
> - 对象总数 **16**（项目 / 设备 / 资产 / 合同等）
> - 三层关联总数 **13,652** 条
> - 底层 EAV 实体 **23.5 万** / EAV 值 **293 万**
>
> 数据来自 **22 个业务域**的 BA/DA/AA 三份架构 Excel，目前已跑通 4 个域（输配电 / 计财 / 产业金融 / 市场营销）的抽取。"

**常被问**：
- Q：这些对象怎么来的？
- A：从三层架构 Excel 里用 SBERT 768 维向量 + 层次聚类 + LLM 命名抽出来的高层概念。

---

### STEP 2 对象三层关联 · 4 分钟（**重点**）

**点**：左侧"对象管理"→ 对象卡片里选 **项目 (OBJ_PROJECT)**

**说**：
> "这里是**甲方 1.md 核心需求**的落地：对象 → 概念实体 / 逻辑实体 / 物理实体 三层关联。
>
> 看右侧面板三列：
> - **紫色 · 概念实体** 6 个（业务场景层）
> - **绿色 · 逻辑实体** 23 个（交互表单层）
> - **橙色 · 物理实体** 23 个（数据库层）
>
> 这是**一个抽象的'项目'对象**穿到三层架构上的全景视图。甲方 3.1 明确说'一个概念实体可以对应多个逻辑实体'，我们严格按这个约束建的关联。"

**常被问**：
- Q：为什么逻辑层数远大于概念层？
- A：一个概念实体下挂多个逻辑实体，一对多关系；我们通过 `via_concept_entity` 字段记录穿透路径。

---

### STEP 3 三层架构知识图谱 · 3 分钟（**视觉冲击**）

**点**：左侧"三层架构图谱"面板 → 深度选 3

**说**：

> - 中心节点 = 抽取对象
> - 三色辐射 = 三层实体
> - 连线粗细 = 关联强度（0.6-0.9 分级）
>
> 节点可拖动、可缩放、可点击展开详情。"

**可选展示**：拖动一个节点给领导看交互，缩放看全局密度。

### STEP 5 全生命周期 · 2 分钟

**点**：左侧"全生命周期管理"→ 选一个对象（比如 项目 或 合同）

**说**：

> 每个对象统一 **4 阶段**：立项 → 设计 → 施工 → 运行。每个阶段展开看该时点的字段快照，支持**时态建模**。
>
> 16 对象 × 4 阶段 = **64 条生命周期记录**，库里全有。"

### STEP 6 溯源 + 机理函数 + 预警 · 2 分钟

**点**：左侧顺序点 **穿透式溯源** → **机理函数** → **预警辅助决策**

**说**（走马观花，每个 30-40 秒）：
> "**溯源**：3 条预置链路 + 15 个节点，展示计财域'结算 → 立项 → 合同 → 施工'的完整穿透。
>
> **机理函数**：3 个预置（阈值 / 公式 / 规则），支持'合同额度 > 300 万走路径 A'这类业务规则的实时求值。
>
> **预警**：基于机理函数扫描 EAV 数据生成预警记录，可直接在界面点'处理'关闭。"

## Q&A 预案（按可能性排序）

### 高可能性

| 问 | 答 |
|---|---|
| 南网服务器内网没外网，怎么办？ | 所有依赖（JDK/Docker/Node/Maven 包/SBERT 模型）都打包离线带过去，无需外网。 |
| DeepSeek 在内网调不了怎么办？ | 接南网内部 LLM 网关（改一个环境变量）；或降级走本地关键词规则命名，不影响主功能。 |
| 达梦和 MySQL 差异多大？ | 建表 DDL 已写达梦专用版，差异在 IDENTITY/COMMENT/关键字上，现场由 DBA 过一遍关键字检查。 |
| 12 个月期满怎么办？ | 过验收则续签或迁正式环境；代码在南网内网仓可直接交接。 |

### 中可能性

| 问 | 答 |
|---|---|
| 抽取准确率多少？ | 当前两域 16 对象 100% 通过 LLM 命名 + 垃圾过滤；定量评估待甲方给黄金标准集。 |
| 新域加进来要多久？ | Excel 放进 DATA/ 就能识别；本地跑一次抽取 ~5 分钟；结果带到服务器刷一次就出数据。 |
| 和企业中台怎么对比？ | 1.md 用"很可能"，不是硬性需求；等中台数据格式明确后加 diff 模块，预计 3-5 天。 |
| 性能上限多少？ | EAV 水平扩展，现已 23 万实体 / 293 万值稳定；达梦侧按分区建索引再扩 10×。 |

### 低可能性（高风险）

| 问 | 答 |
|---|---|
| 能用国产 LLM 替代 DeepSeek 吗？ | 能。只要是 OpenAI 兼容接口，改 `DEEPSEEK_API_BASE` 一项即可。 |
| 有对接 ERP / 物资系统的打算吗？ | 1.md 未明确，按"对象管理器 + 可扩展"框架设计，实际对接需要 ERP 那边给接口规格。 |
| 前端组件库什么时候能换？ | 拿到南网 SDK 当天可换，ElementPlus 占位层已解耦，2-3 人日完成替换。 |

## 演示中的应急

| 场景 | 处理 |
|------|------|
| 浏览器页面白屏 | F12 看 Console；多半是缓存，`Ctrl+F5` 强刷 |
| ECharts 图谱不渲染 | 网络面板看 `/api/olm/graph-data-three-tier` 是否 200；若超时，选深度=2 缩减数据量 |
| 切换域无数据 | 正常 — 该域 outputs JSON 未生成；切回输配电 / 计财继续演示 |
| 某个面板 500 错误 | 不纠结，跳过；演示结束后看 webapp.log 定位 |
| 服务突然挂了 | 另开终端 `bash start.sh --restart`，一般 10 秒内恢复 |
