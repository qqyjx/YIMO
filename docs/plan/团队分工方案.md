# 南方电网智能数据管理项目 — 团队分工方案

> 版本：v1.0 | 日期：2026-03-15 | 编制：项目负责人

---

## 一、项目概述

本项目包含两个子系统，服务于南方电网数字化转型：

### 系统 1：对象管理器（YIMO）

以财务系统为试点，构建底层对象管理器，从企业架构数据中自动抽取核心业务对象（项目、设备、资产等），建立与三层架构（概念实体、逻辑实体、物理实体）的关联关系，实现穿透式业务监管。

- **当前状态**：核心功能已完成，进入维护迭代阶段
- **技术栈**：Python Flask + MySQL + SBERT + ECharts

### 系统 2：函数编排服务（低代码开发平台）

构建面向电网业务场景的函数低代码编排平台，将机理模型、业务规则封装为标准函数服务，通过可视化 DAG 工作流实现业务流程的拖拽式编排与自动执行。

- **当前状态**：从零启动，需完成全部设计与开发
- **技术栈**：Python Flask + MySQL + Vue3 + AntV X6

## 四、协作机制

### 沟通节奏

| 频率 | 形式 | 内容 |
|------|------|------|
| 每周一次 | 周会（线上/线下） | 各自进度汇报、阻塞问题讨论、下周计划 |
| 每日 | 群消息 | 关键进展同步、紧急问题即时沟通 |
| 每月一次 | 里程碑评审 | 交付物验收、计划调整 |

### 代码协作

- **代码仓库**：Git，每个系统一个分支或独立仓库
- **分支规范**：`feature/<功能名>`、`fix/<问题名>`
- **提交规范**：Conventional Commits（`feat:`, `fix:`, `docs:`, `chore:`）
- **代码评审**：关键模块（DAG 引擎、安全相关）须交叉 Review

### 接口对接

- 两系统共用 MySQL 数据库（YIMO 的业务对象数据可被低代码平台引用）
- 负责人在 Phase 1 输出《系统间接口规范》，定义数据交换格式
- YIMO 提供业务对象查询 API，低代码平台调用获取对象元数据
