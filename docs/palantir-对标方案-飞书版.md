# YIMO — 电力行业智能对象管理平台（对标 Palantir Foundry）

> 版本: v1.0 | 日期: 2026-03-15 | 适用场景: 内部汇报

---

## 1. 一页总结

YIMO 是什么？

YIMO 是面向电力行业的对象抽取与三层架构关联系统。它从南方电网的数据架构文档（DA-01/02/03 Excel）中，利用 SBERT 语义聚类 + LLM 归纳命名，自动提炼业务对象（项目、设备、资产、合同等），并建立对象与概念实体→逻辑实体→物理实体的穿透式关联关系。

核心差异化价值：

1. 自动对象抽取：Palantir Foundry 需要人工定义对象类型（耗时数天/数周），YIMO 通过语义聚类算法实现分钟级自动抽取
2. 电力行业深度适配：原生理解 DA-01/02/03 三层架构标准，内置电力术语库
3. 离线部署能力：三级回退机制（SBERT+LLM → SBERT+规则 → 纯规则），支持无外网的内网环境部署

## 2. Palantir Foundry 简介

一句话定位：Palantir Foundry 是全球最大的企业数据操作系统，2025 年收入 >28 亿美元，市值一度超 2500 亿美元。

### 核心概念

| 概念 | 中文释义 | 作用 | 特点 |
|------|----------|------|------|
| Ontology | 本体/对象模型 | 把数据变成"东西"：对象类型→对象→属性→关联 | 需人工定义，耗时数天/数周 |
| Pipeline | 数据管道 | 100+ 预置连接器接入各系统数据 | 支持 SAP/Oracle/MySQL/Kafka |
| Workshop | 工作流编排 | 用对象做事，支持写回源系统 | 核心差异化能力 |
| AIP | AI 平台 | LLM 操作 Ontology，自然语言→查询→操作 | 2023 年后新方向 |

### 为什么大型组织愿意付千万年费

1. 跨系统统一视角：50+ 套业务系统的数据统一到一个操作平台
2. 端到端操作能力：不只是看数据，还能通过 Workshop 直接操作和写回
3. 驻场工程师服务：FDE 团队常驻客户现场，确保落地效果

## 3. YIMO vs Palantir Foundry 对比

### 核心维度对比

| 维度 | Palantir Foundry | YIMO | 评估 |
|------|------------------|------|------|
| 对象建模 | 人工定义（天/周级） | SBERT 自动抽取（分钟级） | ✅ YIMO 优势 |
| 关联类型 | 离散关系 | 连续强度 0~1 + 穿透路径 | ✅ YIMO 优势 |
| 行业适配 | 通用跨行业 | 电力行业深度适配（DA-01/02/03） | ✅ YIMO 优势 |
| 数据接入 | 100+ 预置连接器 | 仅 Excel 导入 | ⚠️ 差距较大 |
| 存储规模 | PB 级分布式 | MySQL 单机 + FAISS | ⚠️ PoC 级别 |
| 工作流引擎 | Workshop + 写回能力 | 3 个预置机理函数 | ⚠️ 脚手架 |
| 可视化 | Quiver/Slate/地图/3D | ECharts 图谱 + Sankey + 治理看板 | ⚠️ 基础可用 |
| AI 能力 | AIP (LLM 操作 Ontology) | SBERT + DeepSeek RAG | ⚠️ 基础可用 |
| 认证授权 | 企业 SSO + RBAC + MFA | 无认证（公开访问） | 🔲 未实现 |
| 审计合规 | 完整操作审计 + 数据血缘 | 无审计日志 | 🔲 未实现 |

### YIMO 的三大结构性优势

#### 优势一：自动对象抽取（核心创新点）

| 对比项 | Palantir Foundry | YIMO |
|--------|------------------|------|
| 定义方式 | 领域专家 + 数据工程师手动梳理 | SBERT 语义聚类 + LLM 自动归纳 |
| 所需时间 | 数天 ~ 数周 | 2-3 分钟 |
| 所需人力 | 需要领域专家参与 | 零人工干预 |
| 实际成果 | — | 输配电域 8 对象/12,282 关联；计财域 7 对象/1,294 关联 |

#### 优势二：电力行业深度适配

- 原生支持南方电网 DA-01/02/03 Excel 标准格式
- 内置电力行业术语的 SBERT 中文专用模型（text2vec-base-chinese，768 维）
- 三层架构关联模型专为概念→逻辑→物理穿透设计
- 多域自动发现（DATA/ 目录添加文件夹即可扩展新域）

#### 优势三：离线部署 + 国产化兼容

| 回退级别 | 条件 | 能力 |
|----------|------|------|
| Level 1 | 有外网 + DeepSeek API | LLM 归纳命名（最优质量） |
| Level 2 | 仅内网 + SBERT 模型 | 关键词规则命名（良好质量） |
| Level 3 | 最小环境 | 纯规则抽取（基础可用） |

Docker Compose 一键部署，无需云服务依赖。

## 5. 演示场景

### 场景 1：输配电域对象抽取 → 三层知识图谱

> 操作：打开 http://localhost:5000 → 选择"输配电"域 → 查看对象卡片 → 点击"三层知识图谱"
>
> 展示：8 个自动抽取的对象（项目、设备、资产等）与 12,282 条三层关联的可视化网络
>
> 亮点：从 3 个 Excel 文件 → 2 分钟自动抽取 → 可交互知识图谱

### 场景 2：跨域重复对象检测 → Sankey 流向图

> 操作：点击"跨域检测" → 查看输配电与计财域的重复对象 → 切换到 Sankey 图
>
> 展示：两个域共享的对象（如"项目"在两域均被抽取） → 数据流向可视化
>
> 亮点：自动发现跨域数据一致性问题

### 场景 3：数据治理看板

> 操作：点击"治理看板" → 查看 8 项治理指标
>
> 展示：三层覆盖率、关联强度分布、缺陷识别（缺少逻辑/物理层关联的对象）
>
> 亮点：自动化数据质量评估，无需人工盘点

## 7. 竞争定位总结

### 我们不是"中国版 Palantir"

| 对比维度 | Palantir Foundry | YIMO |
|----------|------------------|------|
| 定位 | 通用企业数据操作系统 | 电力行业垂直对象管理平台 |
| 客户 | 跨行业（军工、医疗、能源、金融） | 南方电网 + 省级电力公司 |
| 核心场景 | 全场景数据整合与操作 | 数据架构治理 + 三层关联分析 + 跨域对象统一 |
| 核心优势 | 20 年积累 + 千人团队 + PB 级基础设施 | 自动对象抽取（Foundry 依赖人工定义） |
| 成本 | 年费数百万~数千万美元 | 开源 + 定制开发 = Foundry 年费的 1/10 |
| 部署 | 需云/私有云 + FDE 驻场 | Docker Compose 一键部署 + 支持内网离线 |

### 核心竞争力

1. 效率差异：建模时间从"天/周"→"分钟"，降低 80%+ 建模工作量
2. 行业深度：DA-01/02/03 原生适配，电力术语中文专用模型
3. 成本优势：开源代码 + 定制开发模式，Foundry 年费的十分之一
4. 合规优势：全中文 NLP、国产数据库兼容、代码完全自主可控
